dothub / deploy decision

Four ways to make dothub a real remote.

dothub runs fully on localhost right now. To make it a public site a Claude Code session anywhere can hit, you need it on a server with a domain and TLS. Below: what each path costs, takes, and locks you into. Prices are us-east-1, smallest viable config, July 2026, for a low traffic side project (a few users, a few hundred setups).

The short version

If you want the learning outcome with the least lock-in: pick Terraform IaC. You write the infra as code, run one command, and can tear the whole thing down for free in one command. If you just want it live fast and don't care about reproducibility, Real AWS deploy via console is fine. If you're not ready to spend, Just update the runbook costs nothing and leaves the door open.

I write a Terraform module: VPC, EC2, RDS Postgres, S3 bucket, IAM instance role, security groups. Plus an updated runbook. You run terraform apply.

My recommendation: B, Terraform. It directly serves the original "learn to deploy" goal while keeping teardown to one command, so the cost risk is bounded. The infra-as-code skill is the most transferable outcome of the four. If budget is the blocker, C costs nothing and leaves B open for later.
